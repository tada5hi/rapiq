URL Codec ​

@rapiq/codec-url-simple moves a Query over the wire: URLEncoder turns the AST into a URL query string, URLDecoder turns a query string back into the AST.

sh
npm install @rapiq/core @rapiq/parser-simple @rapiq/codec-url-simple

Encoding ​

typescript
import { URLEncoder } from '@rapiq/codec-url-simple';

const encoder = new URLEncoder();
const queryString = encoder.encode(query);
// fields=id,name&filter[age]=>=18&page[limit]=25&include=realm&sort=-age

The output uses the JSON:API-style parameter names (fields, filter, page, include, sort) and the same operator prefixes the simple dialect parses — encode → decode round-trips.

Per-parameter encoders exist too: encodeFields, encodeFilters, encodePagination, encodeRelations, encodeSort.

Decoding ​

typescript
import { URLDecoder } from '@rapiq/codec-url-simple';

const decoder = new URLDecoder();
const query = decoder.decode('filter[age]=>=18&sort=-age&page[limit]=25');

decode(input) parses the raw string (via qs) and returns a Query, or null for non-object input. Per-parameter variants: decodeFields, decodeFilters, decodePagination, decodeRelations, decodeSort.

Decode does not validate

URLDecoder performs no schema validation. When the string comes from an untrusted client, validate it — either parse the qs output with a schema-aware SimpleParser:

typescript
import { parse } from 'qs';
import { SimpleParser } from '@rapiq/parser-simple';

const parser = new SimpleParser(registry);
const query = parser.parse(parse(rawQueryString), { schema: 'user' });

— or rely on a framework that already gives you the parsed object (express's req.query) and skip the decoder entirely.
